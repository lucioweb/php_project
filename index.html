Apache2 Default Page
It works!

Esta página substitui a página padrão de boas-vindas do Apache, usada para testar o operação do servidor Apache2 após a instalação em sistemas Ubuntu. Se você pode ler esta página, isso significa que o servidor Apache foi instalado corretamente em Ubuntu 22.04. Você deve substituir esse arquivo (localizado em /var/www/html/index.html) antes de continuar a operar seu servidor HTTP.

Verifique os caminhos:

$ cd /var/www

$ cd /etc/apache2/sites-available

$ cd /etc/apache2/sites-enabled

Crie em $ cd /etc/apache2/sites-available o virtual host fsphp (por exemplo):$ sudo nano /etc/apache2/sites-available/fsphp.conf, com o conteúdo abaixo:

            <VirtualHost *:80>
                ServerName fsphp
                ServerAlias www.fsphp
                ServerAdmin webmaster@localhost
                DocumentRoot /var/www/fsphp
                ErrorLog ${APACHE_LOG_DIR}/error.log
                CustomLog ${APACHE_LOG_DIR}/access.log combined
            <VirtualHost>
            
              
                .
                ├── apache2.conf
                ├── conf-available
                ├── conf-enabled
                ├── envvars
                ├── magic
                ├── mods-available
                ├── mods-enabled
                ├── ports.conf
                ├── sites-available
                │   ├── 000-default.conf
                │   ├── default-ssl.conf
                │   └── fsphp.conf
                └── sites-enabled
                    └── fsphp.conf -> ../sites-available/fsphp.conf

Ative o host com o comando: $ sudo a2ensite fsphp.conf e reinicie o servidor.

Desative o host default com o comando: $ sudo a2dissite 000-default.conf e reinicie o servidor.
